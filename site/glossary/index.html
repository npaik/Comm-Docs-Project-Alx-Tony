Glossary

Term Definition
brew brew command is a command-line tool for macOS that allows users to install and manage software packages
git config Command-line tool used to set and manage Git configuration options at different levels (system, user, and repository)
repository A central location where Git version control system stores all the data related to a project, including the entire history of changes made to the project over time
git add git add is a Git command used to stage changes made to files in a repository, preparing them to be committed to the repository's history
git commit git commit is a command used in Git version control system to save the changes made to a file or a set of files along with a descriptive message that explains the changes
git push git push is a command used in Git version control system to upload the local repository changes to a remote repository, typically hosted on GitHub
git pull git pull is a command used in Git version control system to merge changes from a remote repository to a local repository, ensuring that the local repository is up to date with the remote repository
git clone git clone is a command used to create a copy of a remote Git repository onto a local machine, allowing developers to work on the codebase without directly modifying the original repository.
branch In Git version control system, a branch is a parallel version of the main codebase that allows developers to work on new features or modifications without directly affecting the main codebase until the changes are ready to be merged
HEAD In Git, HEAD is like a bookmark that points to the current version of your code that you're working on. Whenever you make changes and commit them, HEAD gets updated to point to the latest commit
